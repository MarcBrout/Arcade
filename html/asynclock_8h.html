int bunny_asynclock ( double  elasped_time,
t_bunny_call_order  order 
)

This function manage all traps accordingly to the indicated elapsed_time. It updates the current_time accordingly to what you give to it and call functions. The t_bunny_call_order is a filter allowing to call only a subset of previously registered functions.

/!\ You are NOT supposed to use this function if you use bunny_loop or bunny_loop_mw. These functions already call bunny_asynclock!

Parameters
elapsed_timeThe elasped time since the last call to bunny_asynclock, in seconds. If you wish to use bunny_asynclock by yourself (without bunny_loop* functions), you should use clock_gettime or gettimeofday to get the time progression. In case you use a timeval*, for example, the time in the bunny format will be ((double)tv->tv_sec + (double)tv->tv_nsec / 1e9)
orderAre we before the function set with bunny_loop_main_function or after?
Returns
How many traps are still registered. Useful to stack tasks to end, make a cartoon or things like that.

t_bunny_trap* bunny_new_trap ( t_bunny_trap_function  func,
t_bunny_call_order  order,
double  start_time,
double  duration,
void *  param 
)

Define a new trap: a function that will be called under certain circumstances.

  • - If start_time is positive : - -

If duration is superior to 0:

  • Each loop if the current time is between start_time and start_time + duration

If duration is 0 or really close to 0 (gl_bunny_play_it_once):

  • The function will be called only one time if the current time is after start_time and the function was never called

If duration is negative:

  • Every -duration, the function is called again
  • - If start_time is negative or zero : - -

The trap will wait for every tasks stacked before it to be played. Its behavior will be exacly the same except for the previously specified point.

The function is called with the elapsed time, the returned t_bunny_trap and the sent param as parameter.

Parameters
funcThe function to call under specified conditions
orderThe function will be called
Returns
Return the structure that incarnates the trap. NULL on error.
